# π₀-FAST (Pi0-FAST)

π₀-FAST 是一個**用於通用機器人控制的視覺-語言-動作（Vision-Language-Action）模型**，使用自迴歸的下一個 token 預測來建模連續的機器人動作。

## 模型概覽

π₀-FAST 結合了視覺-語言（Vision-Language）模型的強大能力與一種名為 **FAST（Frequency-space Action Sequence Tokenization）** 的新穎動作標記化方法。這使得自迴歸 VLA 能夠在高度靈巧的任務上進行訓練——這些任務用標準的分箱離散化方法是不可能完成的——同時訓練速度比 π₀ 等基於擴散的方法**快達 5 倍**。

### 為什麼使用 FAST？

標準的機器人動作標記化（tokenization）方法採用簡單的逐維度、逐時間步的分箱方案（binning schemes）。這對簡單行為尚可應付，但在需要精確度與高頻控制的複雜靈巧技能上，效果會迅速崩解。

FAST 透過使用訊號處理技術壓縮動作序列來解決這個問題，產生密集的動作 token 序列，可以像語言 token 一樣進行自迴歸預測。

### FAST Tokenization 的運作方式

FAST 標記器透過以下步驟壓縮動作序列：

1. **正規化（Normalize）**：取一個形狀為 `(H, D)` 的連續動作區塊（chunk），其中 `H` 是時間範圍，`D` 是動作維度。使用支援的正規化方法之一進行正規化（建議使用分位數正規化以處理離群值）。

2. **離散餘弦轉換（Discrete Cosine Transform, DCT）**：對每個動作維度分別套用 DCT（透過 scipy）。DCT 是一種常用於影像和音訊編解碼器（JPEG、MP3）的壓縮演算法。

3. **量化（Quantization）**：對每個動作維度進行四捨五入並移除不重要的係數，產生稀疏的頻率矩陣。

4. **攤平（Flatten）**：將矩陣攤平為一維向量，低頻分量排在前面。

5. **位元組對編碼（Byte Pair Encoding, BPE）**：訓練一個 BPE 標記器來將 DCT 係數壓縮為密集的動作 token，通常可達到比先前標記化方法 **10 倍的壓縮率**。

這種方法可以透過訓練預測這些 FAST token，將**任何現有的 VLM** 轉換為 VLA。

## 訓練自訂 FAST tokenizer

您有兩種 FAST tokenizer 的選擇：

1. **使用預訓練 tokenizer**：`lerobot/fast-action-tokenizer` 標記器是在 100 萬筆以上的真實機器人動作序列上訓練的，可作為通用 tokenizer 使用。

2. **訓練自己的 tokenizer**：為了在您的特定資料集上獲得最佳效能，您可以在自己的資料上微調 tokenizer。

### 訓練您自己的 Tokenizer

```bash
lerobot-train-tokenizer \
    --repo_id "user/my-lerobot-dataset" \
    --action_horizon 10 \
    --encoded_dims "0:6" \
    --vocab_size 1024 \
    --scale 10.0 \
    --normalization_mode QUANTILES \
    --output_dir "./my_fast_tokenizer" \
    --push_to_hub \
    --hub_repo_id "username/my-action-tokenizer"
```

### 主要的 Tokenizer 參數

| 參數                   | 說明                                                                       | 預設值       |
| ---------------------- | -------------------------------------------------------------------------- | ------------ |
| `--repo_id`            | LeRobot 資料集的儲存庫 ID                                                  | 必填         |
| `--action_horizon`     | 每個區塊中的未來動作數量                                                   | `10`         |
| `--encoded_dims`       | 以逗號分隔的編碼維度範圍（例如 `"0:6,7:23"`）                              | `"0:6,7:23"` |
| `--vocab_size`         | BPE 詞彙表大小                                                             | `1024`       |
| `--scale`              | 用於量化的 DCT 縮放因子                                                    | `10.0`       |
| `--normalization_mode` | 正規化模式（`MEAN_STD`、`MIN_MAX`、`QUANTILES`、`QUANTILE10`、`IDENTITY`） | `QUANTILES`  |
| `--sample_fraction`    | 每個 episode 取樣的區塊比例                                                | `0.1`        |

## 使用方式

若要在 LeRobot 中使用 π₀-FAST，請將策略類型指定為：

```python
policy.type=pi0_fast
```

## 訓練

若要訓練 π₀-FAST，您可以使用 LeRobot 訓練腳本：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=pi0_fast \
    --output_dir=./outputs/pi0fast_training \
    --job_name=pi0fast_training \
    --policy.pretrained_path=lerobot/pi0_fast_base \
    --policy.dtype=bfloat16 \
    --policy.gradient_checkpointing=true \
    --policy.chunk_size=10 \
    --policy.n_action_steps=10 \
    --policy.max_action_tokens=256 \
    --steps=100000 \
    --batch_size=4 \
    --policy.device=cuda
```

### 主要的訓練參數

| 參數                                   | 說明                               | 預設值                          |
| -------------------------------------- | ---------------------------------- | ------------------------------- |
| `--policy.gradient_checkpointing=true` | 在訓練期間大幅減少記憶體使用量     | `false`                         |
| `--policy.dtype=bfloat16`              | 使用混合精度訓練以提升效率         | `float32`                       |
| `--policy.chunk_size`                  | 要預測的動作步數（動作時間範圍）   | `50`                            |
| `--policy.n_action_steps`              | 要執行的動作步數                   | `50`                            |
| `--policy.max_action_tokens`           | 每個動作區塊的最大 FAST token 數量 | `256`                           |
| `--policy.action_tokenizer_name`       | 使用的 FAST 標記器                 | `lerobot/fast-action-tokenizer` |
| `--policy.compile_model=true`          | 啟用 torch.compile 以加速訓練      | `false`                         |

## 推論

### 以 KV 快取加速推論

π₀-FAST 支援 **KV 快取（KV-caching）**，這是 LLM 推論中廣泛使用的優化技術。它會快取注意力機制中的鍵值對，避免自迴歸解碼過程中的冗餘計算。

```python
# KV 快取預設為啟用
policy.use_kv_cache=true
```

### 推論範例

```python
from lerobot.policies.pi0_fast import PI0FastPolicy, PI0FastConfig

# 載入策略
policy = PI0FastPolicy.from_pretrained("your-model-path")

# 進行推論
actions = policy.predict_action_chunk(batch)
```

## Model Architecture

π₀-FAST 使用基於 PaliGemma 的架構：

- **視覺編碼器**：SigLIP 視覺塔（vision tower），用於影像理解
- **語言模型**：Gemma 2B，用於處理語言指令和預測動作 token

模型接收影像、文字指令和機器人狀態作為輸入，並輸出離散的 FAST token，再將其解碼回連續動作。

## 配置選項

| 參數                 | 說明                                     | 預設值     |
| -------------------- | ---------------------------------------- | ---------- |
| `paligemma_variant`  | VLM 骨幹變體（`gemma_300m`、`gemma_2b`） | `gemma_2b` |
| `max_state_dim`      | 最大狀態向量維度（填充後）               | `32`       |
| `max_action_dim`     | 最大動作向量維度（填充後）               | `32`       |
| `temperature`        | 取樣溫度（0.0 為貪婪取樣）               | `0.0`      |
| `max_decoding_steps` | 最大解碼步數                             | `256`      |
| `use_kv_cache`       | 啟用 KV 快取以加速推論                   | `true`     |

## 與 π₀ 的比較

| 特性         | π₀             | π₀-FAST              |
| ------------ | -------------- | -------------------- |
| 動作表示方式 | 流匹配（擴散） | 自迴歸 Token（FAST） |
| 訓練速度     | 1x             | **快 5 倍**          |
| 靈巧度       | 高             | 高                   |
| 推論方法     | 迭代去噪       | 自迴歸解碼           |
| KV 快取      | 不適用         | 支援                 |

## 重現 π₀Fast 結果

我們使用 LeRobot 實作在 LIBERO 基準測試上重現了 π₀Fast 的結果。我們取用 LeRobot PiFast 基礎模型 [lerobot/pi0fast-base](https://huggingface.co/lerobot/pi0fast-base)，並以 bfloat16 精度額外微調 40kk 步，批次大小為 256，在 8 張 H100 GPU 上使用 [HuggingFace LIBERO 資料集](https://huggingface.co/datasets/HuggingFaceVLA/libero) 進行訓練。

微調後的模型可在此處找到：

- **π₀Fast LIBERO**: [lerobot/pi0fast-libero](https://huggingface.co/lerobot/pi0fast-libero)

使用以下訓練指令：

```bash
lerobot-train \
  --dataset.repo_id=lerobot/libero \
  --output_dir=outputs/libero_pi0fast \
  --job_name=libero_pi0fast \
  --policy.path=lerobot/pi0fast_base \
  --policy.dtype=bfloat16 \
  --steps=100000 \
  --save_freq=20000 \
  --batch_size=4 \
  --policy.device=cuda \
  --policy.scheduler_warmup_steps=4000 \
  --policy.scheduler_decay_steps=100000 \
  --policy.scheduler_decay_lr=1e-5 \
  --policy.gradient_checkpointing=true \
  --policy.chunk_size=10 \
  --policy.n_action_steps=10 \
  --policy.max_action_tokens=256 \
  --policy.empty_cameras=1 \
```

接著我們使用 LeRobot 的 LIBERO 實作來評估微調後的模型，執行以下指令：

```bash
tasks="libero_object,libero_spatial,libero_goal,libero_10"
lerobot-eval \
  --policy.path=lerobot/pi0fast-libero \
  --policy.max_action_tokens=256 \
  --env.type=libero \
  --policy.gradient_checkpointing=false \
  --env.task=${tasks} \
  --eval.batch_size=1 \
  --eval.n_episodes=1 \
  --rename_map='{"observation.images.image":"observation.images.base_0_rgb","observation.images.image2":"observation.images.left_wrist_0_rgb"}'
```

**注意：** 我們將 `n_action_steps=10`，與原始 OpenPI 實作相同。
