# π₀ (Pi0)

π₀ 是一個**用於通用機器人控制的視覺-語言-動作（Vision-Language-Action）模型**，由 Physical Intelligence 開發。LeRobot 的實作改編自他們的開源 [OpenPI](https://github.com/Physical-Intelligence/openpi) 儲存庫。

## 模型概覽

π₀ 代表了機器人領域的一項突破，是由 [Physical Intelligence](https://www.physicalintelligence.company/blog/pi0) 開發的首個通用型機器人基礎模型。不同於傳統為重複動作編程的狹隘專用機器人程式，π₀ 被設計為一種通用策略，能夠理解視覺輸入、解讀自然語言指令，並控制各種不同的機器人執行多樣化的任務。

### Physical Intelligence 的願景

正如 Physical Intelligence 所描述的，雖然 AI 在數位領域已取得卓越的成就，從下棋到藥物發現皆然，但在物理世界中，人類智慧仍大幅領先 AI。套用莫拉維克悖論（Moravec's paradox）的說法，贏得一盤棋對 AI 而言是「簡單」的問題，但摺一件襯衫或清理桌面卻需要解決一些有史以來最困難的工程問題。π₀ 代表了開發人工物理智慧的第一步，使用戶能夠像使用大型語言模型一樣，簡單地要求機器人執行任何想要的任務。

### 架構與方法

π₀ combines several key innovations:

- **流匹配（Flow Matching）**：使用一種新穎的方法，透過流匹配（擴散模型的一種變體）為預訓練的 VLM 增添連續動作輸出能力
- **跨載體訓練（Cross-Embodiment Training）**：使用來自 8 個不同機器人平台的資料進行訓練，包括 UR5e、雙臂 UR5e、Franka、雙臂 Trossen、雙臂 ARX、移動式 Trossen 和移動式 Fibocom
- **網際網路規模預訓練**：從一個擁有 30 億參數的預訓練視覺-語言模型繼承語義知識
- **高頻率控制**：以高達 50 Hz 的頻率輸出馬達指令，實現即時靈巧操作

# 訓練資料與能力

π₀ 使用迄今為止最大的機器人互動資料集進行訓練，結合了三個關鍵資料來源：

1. **網際網路規模預訓練**：來自網路的視覺-語言（Vision-language）資料，用於語義理解
2. **Open X-Embodiment 資料集**：開源機器人操作資料集
3. **Physical Intelligence 資料集**：涵蓋 8 種不同機器人的大型多樣化靈巧任務資料集

## 使用方式

若要在 LeRobot 中使用 π₀，請將策略類型指定為：

```python
policy.type=pi0
```

## 訓練

要訓練 π₀，您可以使用標準的 LeRobot 訓練腳本搭配適當的設置：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=pi0 \
    --output_dir=./outputs/pi0_training \
    --job_name=pi0_training \
    --policy.pretrained_path=lerobot/pi0_base \
    --policy.repo_id=your_repo_id \
    --policy.compile_model=true \
    --policy.gradient_checkpointing=true \
    --policy.dtype=bfloat16 \
    --policy.freeze_vision_encoder=false \
    --policy.train_expert_only=false \
    --steps=3000 \
    --policy.device=cuda \
    --batch_size=32
```

### 關鍵訓練參數

- **`--policy.compile_model=true`**：啟用模型編譯以加速訓練
- **`--policy.gradient_checkpointing=true`**：大幅減少訓練過程中的記憶體使用量
- **`--policy.dtype=bfloat16`**：使用混合精度訓練以提升效率
- **`--batch_size=32`**：訓練的批次大小，請根據您的 GPU 記憶體調整
- **`--policy.pretrained_path=lerobot/pi0_base`**：您想要微調的 π₀ 基礎模型，選項包括：
  - [lerobot/pi0_base](https://huggingface.co/lerobot/pi0_base)
  - [lerobot/pi0_libero](https://huggingface.co/lerobot/pi0_libero)（專門針對 Libero 資料集訓練）

### 訓練參數說明

| 參數                    | 預設值  | 說明                     |
| ----------------------- | ------- | ------------------------ |
| `freeze_vision_encoder` | `false` | 不凍結視覺編碼器         |
| `train_expert_only`     | `false` | 不凍結 VLM，訓練所有參數 |

**💡 提示**：設定 `train_expert_only=true` 會凍結 VLM，僅訓練動作專家和投影層，讓微調時使用更少的記憶體。

## 相對動作

預設情況下，π₀ 預測絕對動作。您可以啟用**相對動作**，使模型預測相對於當前機器人狀態的偏移量。這對於某些設定可以提升訓練穩定性。

若要使用相對動作，請先透過 CLI 在相對空間中重新計算資料集統計量：

```bash
lerobot-edit-dataset \
    --repo_id your_dataset \
    --operation.type recompute_stats \
    --operation.relative_action true \
    --operation.chunk_size 50 \
    --operation.relative_exclude_joints "['gripper']" \
    --push_to_hub true
```

或等效的 Python 寫法：

```python
from lerobot.datasets.lerobot_dataset import LeRobotDataset
from lerobot.datasets.dataset_tools import recompute_stats

dataset = LeRobotDataset("your_dataset")
recompute_stats(dataset, relative_action=True, chunk_size=50, relative_exclude_joints=["gripper"])
dataset.push_to_hub()
```

`chunk_size` 應與您的策略的 `chunk_size` 一致（π₀ 預設為 50）。`relative_exclude_joints` 列出應保持在絕對空間中的關節名稱（例如夾爪指令）。使用 `--push_to_hub true` 將更新後的統計量上傳至 Hub。

然後啟用相對動作進行訓練：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=pi0 \
    --policy.use_relative_actions=true \
    --policy.relative_exclude_joints='["gripper"]' \
    ...
```
