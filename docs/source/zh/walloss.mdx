# WALL-OSS

WALL-OSS 是由 [XSquare Robot](https://x2robot.com/en/research/68bc2cde8497d7f238dde690) 團隊於 2025 年提出的具身智慧開源基礎模型（foundation model）。LeRobot 的實作改編自其開源的 [WallX](https://github.com/X-Square-Robot/wall-x) 儲存庫。

X Square Robot 的 WALL-OSS 現已整合至 Hugging Face 的 LeRobot 生態系統中。這是 LeRobot 與 X Square Robot 團隊之間一個令人振奮的合作專案。您現在可以直接透過 LeRobot 對 WALL-OSS 進行後訓練、評估和部署。我們的目標是讓開源機器人社群能更輕鬆地客製化與部署 WALL-OSS 基礎模型。歡迎閱讀與探索 WALL-OSS 的[論文](https://arxiv.org/pdf/2509.11766)和[程式碼](https://github.com/X-Square-Robot/wall-x)。

## 模型概覽

WALL-OSS 團隊致力於打造具身基礎模型，以擷取並壓縮世界上最有價值的資料：連續、高保真的物理互動串流。透過在模型決策與實體親身體驗之間建立直接的回饋迴路，使真正可泛化的智慧得以湧現——這種智慧不僅理解世界如何運作，更知道如何在其中有效地行動。

技術上，WALL-OSS 引入了一種緊耦合的多模態架構（緊耦合 MoE 結構），整合了離散與連續動作建模策略。透過兩階段訓練流程（啟發 → 整合），模型逐步統一語義推理與高頻動作生成。其核心創新包括：

- **具身感知增強的多模態預訓練**：在統一的視覺-語言-動作資料上進行大規模訓練，以強化空間、因果和操作理解能力。
- **統一跨層級思維鏈（Uni-CoT）**：單一可微分框架，統一高層指令推理、子任務分解與細粒度動作合成，形成從「理解」到「執行」的連續鏈路。
- **混合專家動作頭（MoE action heads）**：根據任務階段動態啟用專家，在離散或連續空間中建模動作，以維持穩定的 VLM 先驗。
- **兩階段訓練範式**：
  - **啟發階段**：注入離散動作先驗，以強化空間理解和語義-動作對齊。
  - **整合階段**：使用流匹配實現高頻連續控制。

## 安裝需求

1. 按照我們的[安裝指南](./installation)安裝 LeRobot。
2. 執行以下命令安裝 WallX 相依套件：

   ```bash
   pip install -e ".[wallx]"
   ```

## 使用方式

若要在 LeRobot 中使用 WallX，請將策略類型指定為：

```python
policy.type=wall_x
```

## 訓練

若要訓練 WallX，您可以使用標準的 LeRobot 訓練腳本搭配適當的設定：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=wall_x \
    --output_dir=./outputs/wallx_training \
    --job_name=wallx_training \
    --policy.repo_id=your_repo_id \
    --policy.pretrained_name_or_path=x-square-robot/wall-oss-flow \
    --policy.prediction_mode=diffusion \
    --policy.attn_implementation=eager \
    --steps=3000 \
    --policy.device=cuda \
    --batch_size=32
```

### 訓練參數

| 參數                           | 說明                                                                                                       |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------- | --- |
| `--dataset.repo_id`            | 您訓練資料集的 Hugging Face Hub 儲存庫 ID（例如 `lerobot/aloha_sim_insertion_human`）                      |
| `--policy.type`                | 指定使用 WallX 策略架構                                                                                    |
| `--output_dir`                 | 儲存訓練檢查點和日誌的本地目錄                                                                             |
| `--job_name`                   | 此訓練回合的名稱識別碼（用於日誌記錄/追蹤）                                                                |
| `--policy.repo_id`             | 您的 Hugging Face Hub 儲存庫 ID，訓練完成的模型將推送至此                                                  |
| `--policy.pretrained_path`     | 用於初始化的預訓練 WallX 權重路徑（官方 WALL-OSS 檢查點）                                                  |
| `--policy.prediction_mode`     | 動作預測策略：`diffusion` 或 `fast` - `diffusion` 使用迭代去噪進行動作生成，`fast` 則使用下一個 token 預測 |
| `--policy.attn_implementation` | 注意力機制實作後端 - `eager` 使用標準 PyTorch 注意力（其他選項包括 `flash_attention_2` 或 `sdpa`）         |
| `--steps`                      | 要執行的訓練步數總計                                                                                       |
| `--policy.device`              | 訓練裝置（`cuda` 為 GPU，`cpu` 為 CPU）                                                                    |
| `--batch_size`                 | 每個訓練批次的樣本數量                                                                                     |     |
